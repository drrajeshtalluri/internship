A Guide to Building Your First Interactive Python Game (Logic & Concepts)

üéØ Project Goal

To create a Python program where:

  1. The computer randomly picks a secret number within a certain range (e.g., between 1 and 100).
  2. The player gets to guess the number.
  3. The program tells the player if their guess is too high, too low, or correct.
  4. The game continues until the player guesses the secret number.
  5. The program might also count how many guesses it took.

üß† What You'll Learn (Key Concepts)

  • Variables: Storing changing information.
  • User Input: Getting information from the player.
  • Random Number Generation: Making the computer pick a number.
  • Loops (while loops): Repeating code.
  • Conditional Logic (if, elif, else): Making decisions.
  • Data Type Conversion: Changing text to numbers.
  • Output: Displaying messages.
  • Functions (optional but good practice): Organizing code.
  • Error Handling (optional but good practice): Managing invalid input.

üõ†Ô∏è Project Setup with `uv` & Structuring Your Code

Let's set this up as a proper project from the start. This is how professional developers organize their work!

  1. Create a Project Folder:
    • Open your WSL terminal. Navigate to where you keep your coding projects (e.g., cd ~/python-internship).
    • Create a new folder for this game: mkdir number_guessing_game
    • Go into that folder: cd number_guessing_game
  2. Initialize the Project with `uv`:
    • This command creates a pyproject.toml file. This file acts like a configuration or recipe card for your Python project, keeping track of settings and any special Python packages (libraries) your project might need later on.
    • In your terminal, run: uv init
    • uv will ask you a few questions (like project name, Python version). You can usually accept the defaults by pressing Enter.
  3. Create a Virtual Environment with `uv`:
    • A virtual environment is like a private, isolated workspace or sandbox just for this specific project. It keeps all the Python tools and packages for this game separate from other projects and from your main Python installation on your computer. This is super important to avoid conflicts and keep things tidy!
    • In your terminal, run: uv venv
    • This command will create a new folder, usually named .venv, inside your number_guessing_game directory. This .venv folder contains a copy of Python and is where any packages you install for this project will go.
  4. Activate the Virtual Environment:
    • Before you start writing code or installing any Python packages for your game, you need to "activate" this virtual environment. Activating it tells your terminal to use the Python and tools from your project's special .venv sandbox.
    • In your terminal, run: source .venv/bin/activate
    • When the environment is active, your terminal prompt will usually change (often by adding (.venv) at the beginning). This shows you're now working inside the project's isolated environment.
    • Important: You need to run this activation command every time you open a new terminal window to work on this specific project.
  5. Create Your Game File:
    • Now, inside your number_guessing_game folder (and with your virtual environment active), you can create your Python file for the game. Let's call it guesser.py.
    • You can create this file using VS Code (File > New File), or in the terminal with: touch guesser.py
    • To open the entire project folder in VS Code (recommended), make sure you're in the number_guessing_game directory in your terminal, then type: code . (that's "code" space "dot").

Your project structure should now look something like this:

ü§î Let's Plan It Out (The "Algorithm")

Before you write any code in guesser.py, it's crucial to think about the steps your game needs to follow:

  1. Game Initialization:
    • Display a welcome message.
    • Explain the rules.
  2. Computer's Turn:
    • Randomly select a whole number within a range (e.g., 1 to 100). Store it.
  3. Game Loop (Player's Turn):
    • Start a loop that continues until the player guesses correctly.
    • Inside the loop:
      • Ask for and read the player's guess.
      • Convert the input to a whole number.
      • (Optional: Increment guess counter.)
      • Compare guess to secret: too low, too high, or correct. Provide feedback. If correct, end loop.
  4. Game End:
    • (Optional: Ask to play again.)

üíæ Saving Your Work with Git & GitHub

After developing your game locally, save it to GitHub. This is essential for backing up your work, tracking changes, and potentially collaborating or sharing.

  1. Create a GitHub Repository:
    • Go to [GitHub.com](https://github.com/) and log in.
    • Click the "+" icon (top right) ‚Üí "New repository".
    • Name it (e.g., `python-number-guesser`). Add a description. Choose Public/Private.
    • Important: Do NOT initialize with a README, .gitignore, or license. Click "Create repository".
    • Copy the repository URL (HTTPS or SSH) provided by GitHub.
  2. Initialize Git Locally:
    • In your WSL terminal, ensure you're in your number_guessing_game project directory.
    • Run: git init
  3. Create a `.gitignore` file:
    • This tells Git to ignore certain files/folders (like your virtual environment, which shouldn't be tracked).
    • In your project root (number_guessing_game/), create a file named .gitignore.
    • Add this line to it: .venv/
    • Also good to add: __pycache__/ and *.pyc
    • Save the .gitignore file.
  4. Add and Commit Your Files:
    • Stage all your project files for tracking: git add .
    • Commit (save a snapshot) your changes: git commit -m "Initial version of number guessing game"
  5. Link to GitHub & Push:
    • Connect your local repo to the remote GitHub repo: git remote add origin YOUR_GITHUB_REPO_URL_HERE
    • Push your committed code to GitHub: git push -u origin main (or master).
    • Refresh your GitHub repository page to see your files.

For future changes: git add . ‚Üí git commit -m "Describe changes" ‚Üí git push.
